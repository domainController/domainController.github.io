========= DOM XSS =========

  1. "DOM-based XSS can be seen as an information flow problem where portions of strings — potentially under attacker’s control (e.g., URL, cookie value) — are being evaluated as code through JavaScript code evaluation functions like eval() or being used in unsafe dynamic DOM constructions, such as via document.write or innerHTML" (DexterJS)
  2. "DOM-based XSS, which is a subtype of Cross-site Scripting (XSS) problems that results from unsafe processing of untrusted data from sources controlled by the attacker." (DexterJS)

========= Exploit =========

  1. "Exploit = BreakOutSequence + AttackVector + EscapeSequence" (DexterJS)

========= Source =========

  1. "the source locations of incoming data" (DomXssMicro)

========= Propagation =========

  1. "the statements which propagate tainted values from the source to the sink without modifying them" (DomXssMicro)
