  2. "At the core of the presented attack are so-called script gadgets, small fragments of JavaScript contained in the vulnerable site’s legitimate code. Generally speaking, a script gadget is piece of JavaScript code which reacts to the presence of specifically formed DOM content in the Web document. In a gadget-based attack, the adversary injects apparently harmless HTML markup into the vulnerable Web page. Since the injected content does not carry directly executable script code, it is ignored by the current generation of XSS mitigations. However, during the web application lifetime, the site’s script gadgets pick up the injected content and involuntarily transform its payload into executable code. Thus, script gadgets introduce the practice of code-reuse attacks [27], comparable to return-to-libc , to the Web"
