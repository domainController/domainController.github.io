"Cross-origin messages sent via postMessage are “authenticated” in the sense that Web browsers correctly set their origin attribute to the sender’s origin, but the recipient must check this attribute and verify that the message comes from the expected sender. These checks are non-trivial and cross-document messaging is considered the top HTML5 security threat [23]."
  4. "If the thirdparty frame does not contain any security-critical functionality, there is no immediate threat. Unfortunately, our study shows that some popular third-party frames contain unprotected operations such as writing to local storage or cookies which can be triggered by received messages."
